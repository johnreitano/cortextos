# Ticket Triage

1. Pull configured support inboxes.
2. Classify by priority, category, sentiment, customer, and source.
3. Search docs/KB before drafting.
4. Draft reply with citations/internal source links.
5. Escalate bugs, billing, security, legal, account access, or policy exceptions.
6. Log ticket summary and gaps under `support/`.
7. Do not send externally without approval unless setup grants a narrow exception.
