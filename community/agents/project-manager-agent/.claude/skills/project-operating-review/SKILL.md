# Project Operating Review

1. Read `projects/project-registry.json`.
2. Check cortextOS tasks, configured external task tools, approvals, inbox, and blockers.
3. Identify stale, blocked, ownerless, and deadline-risk work.
4. Route specialist work with bus messages or tasks.
5. Produce `projects/reports/YYYY-MM-DD-review.md`.

Report format: Today, Risks, Blockers, Decisions Needed, Handoffs, Next Actions.
